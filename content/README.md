# Контент CS Major (IEM Cologne Major 2026) — как править (двуязычно RU/EN)

Весь сайт генерится из этой папки. Логики тут нет — только данные. Сайт двуязычный: текстовые
поля хранятся как `{ "ru": "...", "en": "..." }`. После любой правки `npm run build` сам прогонит
`npm run validate` и не даст собрать сломанный/неполный контент.

```
content/
  matches.json         — матчи (команды, дата, время МСК, стадион, счёт по картам)
  bets.json            — ставки (Миша / AI, светофор, кэф, статус)
  news.json            — лента справа
  facts.json           — блок «Факты» на главной (массив {ru,en})
  analysis/<id>.md     — разбор матча (RU markdown), имя файла = id матча
  analysis/<id>.en.md  — тот же разбор по-английски (обязателен)
```

## Двуязычность — правило

- **Каждое видимое текстовое поле = `{ "ru": "…", "en": "…" }`.** Оба языка обязательны (валидатор проверяет).
- **Языко-нейтральные поля НЕ переводятся:** `id`, `date`, `timeMsk`, `cc`, `odds`, `tier`, `status`,
  `side` (всегда ключ `"Миша"`/`"AI"` — на экране сам станет Misha/AI), `type` (коды рынка ТС/Фкарт/…),
  `result` (счёт по картам «2:1»), `time` в ленте (ручной МСК-штамп).
- **Время** вводим как `timeMsk` (Москва, UTC+3) — сайт сам пересчитает в зону пользователя.

## Команды — `lib/teams.js`

`cc` в матче — это пара КЛЮЧЕЙ команд из реестра `lib/teams.js`, напр. `["g2","spirit"]`.
Бейдж рисуется из аббревиатуры и цвета бренда (без внешних картинок). Нет команды в реестре —
добавь в `lib/teams.js` (фолбэк всё равно нарисует первые буквы кода).

## Матч → `matches.json`

```json
{
  "id": "g2-spirit",
  "date": "2026-06-19",
  "timeMsk": "16:00",
  "cc": ["g2", "spirit"],
  "result": null,
  "title":  { "ru": "G2 — Spirit", "en": "G2 — Spirit" },
  "stage":  { "ru": "IEM Cologne Major 2026 · BO3", "en": "IEM Cologne Major 2026 · BO3" },
  "venue":  { "ru": "Lanxess Arena, Кёльн", "en": "Lanxess Arena, Cologne" },
  "lede":   { "ru": "Короткая выжимка.", "en": "Short teaser." }
}
```
`venue`, `weather`, `lede` — необязательны (но если есть — двуязычны). `result: "2:1"` (счёт по картам, хозяева:гости) после серии. `cc` — ровно 2 ключа команд.

## Ставка → `bets.json`

```json
{
  "id": 1,
  "date": "2026-06-19",
  "matchId": "g2-spirit",
  "type": "ТС",
  "odds": 3.1,
  "tier": "red",
  "side": "Миша",
  "status": "pending",
  "match": { "ru": "G2 — Spirit", "en": "G2 — Spirit" },
  "bet":   { "ru": "Spirit 2:1 по картам", "en": "Spirit to win 2:1 on maps" },
  "note":  { "ru": "…", "en": "…" }
}
```
`side` — `"Миша"` или `"AI"`. `tier` — `green`/`yellow`/`red` (светофор, размер ставки). `status` — `pending`/`win`/`lose`/`void`.
Системы/экспрессы — массив `legs`, у каждой ноги `m` (матч) и `p` (пик) двуязычны.

**Коды рынков `type` (CS):** `П1`/`П2` (победа в серии), `Фкарт` (фора по картам), `ТС` (точный счёт по картам),
`ТБкарт`/`ТМкарт` (тотал карт), `Фраунд` (фора по раундам), `ТБр`/`ТМр` (тотал раундов на карте),
`Карта1` (рынок 1-й карты), `Пистолет`, `Овертайм`, `Игрок`, `Экспресс`, `Система`, `Другое`. Полные подсказки — `lib/glossary.js`.

## Лента → `news.json`

```json
{ "time": "19.06 · 16:14", "tag": "misha", "text": { "ru": "…", "en": "…" } }
```
`tag`: `lineup` · `insight` · `misha` · `result`. `time` — ручной МСК-штамп. Новые — сверху.

## Факты → `facts.json`

Массив `{ru,en}`. Первый — самый важный.

## Разбор матча → `analysis/<id>.md` (+ `<id>.en.md`)

Обычный markdown, имя файла = `id` из `matches.json`. **Нужны оба файла** (валидатор требует перевод).

## Проверка

```bash
npm run validate    # проверит схему {ru,en}, стороны Миша/AI, теги, наличие <id>.en.md
npm run build       # прогонит validate и соберёт статический сайт в out/
```
